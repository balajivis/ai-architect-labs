# Swarm Architecture

A swarm is a flatter multi-agent design where peer agents collaborate and pass control among themselves, rather than reporting up to a single supervisor. Coordination emerges from the agents' interactions instead of a central boss.

## Key ideas
- Agents act as peers and hand off to whoever is best suited next.
- There is no single central controller directing every step.
- Useful when tasks shift fluidly between specialties.
- More flexible but can be harder to predict and trace.
- Contrasts with the hierarchical supervisor pattern.
