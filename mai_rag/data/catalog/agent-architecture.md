# Agent Architecture

An AI agent is a system that uses a language model as a reasoning engine to decide what actions to take, observe the results, and keep going until a goal is met. Agent architecture is the overall blueprint that connects the model, its tools, its memory, and its control loop into something that can act, not just answer.

## Key ideas
- An agent loops: perceive a situation, decide, act through a tool, observe the outcome, repeat.
- Core building blocks are a reasoning model, a set of tools, some memory, and a controller that manages the loop.
- Architectures range from a single agent with a few tools to teams of specialized agents.
- Good architecture separates "what to do" (reasoning) from "how to do it" (tools and execution).
- Reliability, cost, and safety guardrails are design concerns, not afterthoughts.
