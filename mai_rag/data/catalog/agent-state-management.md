# State Management

State is everything an agent needs to remember while working on a task — the goal, what it has tried, intermediate results, and where it is in a workflow. State management is how that information is tracked, passed along, and updated as the agent runs.

## Key ideas
- State holds the task's progress and context between steps.
- Clear state makes multi-step workflows reliable and resumable.
- State can be passed explicitly between steps or stored centrally.
- Well-managed state helps with debugging and observability.
- It is distinct from long-term memory, which spans sessions.
