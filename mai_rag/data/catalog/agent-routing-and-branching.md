# Routing and Branching

Routing and branching let an agent system take different paths depending on the situation. Instead of one fixed sequence, the system can send a request to the right handler or choose between alternative next steps.

## Key ideas
- Routing directs a task to the most suitable tool, agent, or path.
- Branching chooses between alternatives based on conditions or model judgment.
- This makes systems flexible enough to handle varied inputs.
- A router can classify intent and dispatch accordingly.
- Clear paths keep complex flows understandable and testable.
