# Checkpoints

A checkpoint is a saved snapshot of an agent's state at a point in a workflow. Checkpoints let a long-running agent pause, resume, recover from failures, or hand control to a human without starting over.

## Key ideas
- Checkpoints capture progress so work isn't lost on interruption.
- They enable resuming a task exactly where it left off.
- They support pausing for human review, then continuing.
- They make long or expensive workflows more robust.
- They pair naturally with state management and human-in-the-loop steps.
