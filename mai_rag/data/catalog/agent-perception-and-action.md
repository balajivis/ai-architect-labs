# Perception and Action

Perception and action are the two halves of an agent's loop. Perception is how the agent takes in the current state of its world — inputs, tool results, documents — and action is how it changes that world by doing something.

## Key ideas
- Perception turns raw inputs into something the agent can reason about.
- Action is any step that affects the outside world or moves the task forward.
- The two alternate: the agent perceives, acts, then perceives the result.
- Richer perception (more context) usually leads to better actions.
- Clear boundaries on actions keep agents safe and predictable.
