# Inference

Inference is the act of running a trained model to produce an output, as opposed to training, which builds the model in the first place. Every time you send a prompt and get a response, you are paying for inference.

## Key ideas
- Inference is using a model; training is building it.
- It is where ongoing cost and latency live in production.
- Inference can run in the cloud or, for smaller models, locally.
- Throughput and response speed are key operational concerns.
- Most product economics hinge on inference cost per request.
