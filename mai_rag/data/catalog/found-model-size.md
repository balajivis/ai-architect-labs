# Model Size (7B / 70B / 405B)

Model size is often described by parameter count, such as 7B, 70B, or 405B, where "B" means billions of parameters. Larger models tend to be more capable but cost more to run and need more hardware.

## Key ideas
- Parameter count is a rough proxy for capability and resource needs.
- Smaller models (e.g. 7B) are cheaper and easier to deploy.
- Larger models (e.g. 405B) can handle harder tasks but cost more.
- Size is not the only factor; training data and tuning matter too.
- Picking a size is a balance of quality, latency, and budget.
