# Vector Databases

A vector database stores high-dimensional embeddings and lets you find items that are *semantically* close to a query, not just textually identical. It is the storage and search engine that makes retrieval in RAG fast at scale.

## Key ideas
- Stores embeddings plus metadata, enabling similarity search over large collections.
- Uses approximate nearest-neighbor indexing to stay fast as data grows.
- Supports metadata filtering to combine semantic search with structured constraints.
- Examples in the ecosystem include both dedicated stores and database extensions.
- Choice depends on scale, latency, hosting, and operational needs.
