# Knowledge Base Design

Knowledge base design is the upfront work of organizing source content so a RAG system can retrieve from it well — deciding what to include, how to structure it, and how to keep it fresh. A well-designed knowledge base often matters more than the model choice.

## Key ideas
- Curate authoritative, non-conflicting sources.
- Structure content so the right unit can be retrieved.
- Plan for freshness, updates, and deprecation of stale docs.
- Attach useful metadata for filtering and routing.
- Garbage or contradictory content degrades every downstream answer.
