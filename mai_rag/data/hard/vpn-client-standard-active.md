# Northwind Remote Access — Approved VPN Client (Standard)

This standard defines the single approved virtual private network (VPN) client and the connection configuration that Northwind Technologies employees must use to reach the internal corporate network from outside the office. It applies to the Standard remote-access tier and replaces the previously published version of this standard. The numbers in this document are the values currently in force; where an older edition is still cached or printed, this active edition's figures take precedence. IT Security owns this document and reviews it on a rolling quarterly basis.

## Scope — corporate-issued laptops connecting to Northwind internal network

This standard covers corporate-issued laptops that connect to the Northwind internal network over the public internet. It governs the client software, authentication, posture requirements, and session behavior for those endpoints. Personal devices, contractor-owned hardware, mobile phones, and tablets are out of scope and are handled by separate bring-your-own-device guidance.

All managed laptops issued by Northwind IT are enrolled in the always-on remote-access profile. When a laptop is outside the corporate network, the VPN tunnel is established automatically before user network traffic is permitted. Users may not disable the always-on behavior; doing so is a policy violation and is reported by endpoint telemetry.

## Approved Client (Cisco Secure Client 5.1.4)

The only approved VPN client for the Standard tier is **Cisco Secure Client 5.1.4** (formerly branded Cisco AnyConnect). No other VPN client is permitted on a managed laptop, and unmanaged VPN software must not be installed alongside the approved client. This 5.1.4 build is the current approved version; any earlier edition of this standard that named a different build is no longer in force.

Cisco Secure Client 5.1.4 is distributed and updated through the managed software catalog. Employees should not download the installer from third-party sources. The client connects to the corporate **Cisco Firepower 2130** concentrator, which terminates all Standard-tier remote-access tunnels and enforces the central policy described in this document. Endpoints that present an outdated client version are blocked at the concentrator and prompted to upgrade through the software catalog before access is granted.

Always-on enforcement is delivered through the Cisco Secure Client profile that ships with the managed image. The profile pins the connection to the Firepower 2130 concentrator and re-establishes the tunnel automatically after network changes, sleep/wake cycles, or transient connectivity loss.

## Authentication (multi-factor, device-health gate)

Authentication to the Standard remote-access service requires two factors. The first factor is the user's standard corporate directory credential. The second factor is **Duo MFA**; users approve a push notification or enter a passcode generated by the Duo application during connection setup.

In addition to user authentication, each endpoint must clear a device-health gate before the tunnel is allowed to carry traffic. This gate confirms that the machine is a managed Northwind laptop and that its required protection software is present and running. Specifically, it requires the **CrowdStrike Falcon agent at version 7.x or later** to be installed and reporting as healthy. The Falcon agent is Northwind's endpoint anti-malware and threat-detection layer; it is what stands in for traditional antivirus and endpoint-protection (EDR) requirements on managed laptops. Machines that are missing this agent, or that run a build below 7.x, are turned away until remediated.

A valid device certificate, provisioned during enrollment, identifies the laptop to the concentrator. The combination of the device certificate, Duo MFA, and the CrowdStrike device-health gate forms the authentication and trust baseline for the Standard tier.

## Session Behavior and Limits

The following session controls are enforced centrally at the Cisco Firepower 2130 concentrator and apply to every Standard-tier connection. They are not configurable by end users; IT Security sets them and may tighten them for higher-sensitivity tiers under separate standards.

- **When a connection goes quiet:** if a session carries no traffic for **30 minutes**, the concentrator tears the tunnel down. The user must reconnect and re-authenticate, including completing Duo MFA, before access resumes. This is the inactivity cutoff for the Standard tier.
- **Hard ceiling on a single connection:** regardless of how active it is, one session may stay established for at most **12 hours**. At that point the concentrator forces a disconnect and the user must start a fresh session.

These are the figures in force for the current edition of this standard; if a superseded copy lists different durations, this active edition governs.

## Split Tunneling Policy

**Split tunneling is disabled.** When the VPN is connected, all endpoint network traffic is routed through the Northwind tunnel and the Firepower 2130 concentrator. Traffic destined for the public internet is not permitted to bypass the tunnel.

This full-tunnel posture ensures that web traffic, software-as-a-service access, and DNS resolution are subject to corporate inspection and egress controls while the laptop is connected remotely. Requests to enable split tunneling for performance reasons are not granted under the Standard tier; teams with a documented operational need must seek an exception through IT Security governance.

## Deprecated Clients (Juniper Pulse Secure 9.1 removed)

The previously approved client, **Juniper Pulse Secure 9.1**, is deprecated and has been removed from the environment. It must not be installed or used to reach the Northwind internal network. The legacy Juniper concentrator that served the Pulse Secure client has been decommissioned, and any remaining Pulse Secure installations on managed laptops are removed automatically by endpoint management.

If a laptop still has Juniper Pulse Secure 9.1 present, the user should open the managed software catalog to install Cisco Secure Client 5.1.4 and allow the management agent to remove the obsolete client. No remote-access connectivity is available through the deprecated client.

## Support and Escalation

Employees who cannot connect with Cisco Secure Client 5.1.4 should first confirm that the managed software catalog shows the client as up to date and that the CrowdStrike Falcon agent is running at version 7.x or later. Most connection failures trace back to the device-health gate and resolve once the Falcon agent is healthy.

For Duo MFA enrollment issues, lost devices, or repeated push failures, contact the IT Service Desk, which can reset and re-enroll a second factor. For persistent tunnel failures, questions about why a connection dropped after going quiet or after a long stretch, or suspected concentrator issues on the Firepower 2130, the Service Desk escalates to the IT Security remote-access team. Suspected security incidents — including unexpected prompts, unrecognized connection attempts, or signs of tampering with the client — must be reported to IT Security immediately through the standard incident channel.
